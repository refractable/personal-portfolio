Introduction

Git is a powerful version control system that enables you to track changes to your code, collaborate with others, and revert to previous versions when needed. This guide will introduce you to the fundamental Git commands to get you started.

Prerequisites

You should have Git installed on your system. If not, download it from the official website

git init

Initializes a new Git repository in the current directory.

git init
git clone

Creates a local copy of an existing remote repository.

git clone https://github.com/user/my-project.git
git status

Shows the status of the current working directory (new files, changes, files ready to be committed).

git status
git add

Adds changes to the staging area in preparation for a commit.

git add index.html

(add a single file)

git add .

(add all changes)

git commit

Records staged changes with a descriptive message.

git commit -m "Updated feature description"
git push

Sends local commits to the remote repository.

git push origin master

(pushes to branch 'master')

git pull

Fetches updates from the remote repository and merges them into your local branch.

git pull origin master

    • Important Notes
  • Before making changes, ensure you're on the correct branch.
  • Commit frequently with clear and concise commit messages.
  • Resolve any merge conflicts that might occur during a git pull.
